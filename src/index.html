Moment 1 - Automatisering & publicering

Information om uppgiften

Syfte

Målen med uppgiften är att:

  • Få en inblick hur NodeJS kan användas för att underlätta arbetet med att utveckla en webbplats.
  • Kunna automatisera uppgifter med något kommandobaserat verktyg i NodeJs, med syfte att förenkla utvecklingsarbetet av en webbplats.
  • Kunna skapa en automatiserad utvecklingsmiljö baserad på Git och NodeJs för att kunna göra automatisk publicering vid uppdatering av webbplatsen/Git-repot.
  • Kunna automatisera transpilering och minifiering av CSS, HTML och JavaScript samt automatiskt skapa optimerade grafikfiler.
  • Skapa en så kallad "boilerplate" för webbutveckling, som kan användas som grund för framtida projekt.

Uppgifter

Git och Github

Git och GitHub ska användas regelbundet med bra versionshantering.
Regelbundna commits med tydliga meddelanden krävs.
.gitignore ska användas för att undvika versionering av irrelevanta filer, t.ex. node_modules.
Arbeta med branches, exempelvis dev och main, för att organisera utvecklingsflödet.
Repositoriet ska vara lätt att klona och återanvända för framtida projekt.

Automatisering och publicering

Repositoriet ska göras publikt vid inlämning.
Skapa en webbstruktur med HTML, CSS, JavaScript och bilder.
Använd Node.js för att automatisera uppgifter (t.ex. Parcel eller Gulp).
Paket och beroenden ska listas i package.json, och applikationen ska kunna återställas med npm install.

Automatiserade uppgifter:

  • Minifiering av CSS
  • Transpilering och minifiering av JavaScript
  • Minifiering av HTML
  • Optimering av bilder (automatiskt vid uppdatering)
  • Bakåtkompatibilitet för webbläsare
  • Kopiering av arbetsfiler till en publiceringsmapp

Skapa demowebbplats

Webbplatsen ska ha en responsiv design och innehålla HTML, CSS och JavaScript.
Ska inkludera en huvudmeny med stöd för valfritt antal alternativ.

Minst tre undersidor ska finnas:

  1. Startsida med info om uppgiften
  2. Undersida med minst fem fotografier
  3. Undersida där arbetsprocessen beskrivs

Strukturerad filhantering med separata kataloger för CSS, JavaScript och bilder. Tydlig uppdelning mellan arbetsfiler och publiceringsfiler.

Publicering

Webbplatsen ska publiceras på en host utan FTP/SFTP (t.ex. Netlify, Render, Vercel eller GitHub Pages). Publiceringen ska ske automatiskt vid uppdateringar i GitHub-repot.
Använd branches för att separera utveckling och publicering, t.ex. development för testning och main för produktionsversionen.
Skapa en undersida som detaljerat beskriver arbetsprocessen, vad som automatiserats och vilka verktyg som använts.
